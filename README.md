# Go To place
## URL
[https://gone-place.com/](https://gone-place.com/)
## アプリ概要
- 行きたい場所のgooglemapでの情報(住所,評価,公式サイト)を登録し一覧で確認が出来るアプリです。
- 訪問した場所はレビューとスコアをつけて登録出来ます。
- 訪問してよかった場所はワンクリックで、もう一度行きたいリストに追加出来ます。
- 予定を作成し行きたい場所を予定に追加出来ます。
- 訪問済みの場所をおすすめとして公開できます。また、他の人のおすすめは行きたい場所に登録出来ます。<br>注）***おすすめへの投稿ボタンは訪問済み場所の詳細画面にあります***

## 機能一覧
| 機能名 | 説明 |
|-----|-----|
|ログイン機能|サインアップ・サイインイン・ログアウト|
| ゲストログイン機能|ワンクリックでログインして機能を試せます。|
| 行きたい場所登録|サイト内のmapで検索しgooglemapでの情報(住所,評価,公式サイト)を取得し保存できます。|
| 訪問済み機能 |登録済みの行きたい場所をレビューとスコアをつけて訪問済みに登録出来ます。<br>行きたい場所に登録していなくてもmapで検索し訪問済みに登録することが可能です。| 
| もう一度行きたい登録機能 |訪問済みでもう一度行きたい場所はワンクリックでもう一度行きたいに登録出来ます。<br>解除もワンクリックで可能です。| 
| 予定作成機能 |名前とカラーリングを付けサイト内のカレンダーに追加出来ます。<br>作成した予定には、行きたい場所、もう一度行きたいに登録している場所を時刻を指定して追加出来ます。<br>未登録の場所もmapで検索し追加が可能です。 | 
|おすすめ公開機能|訪問済みの場所は、おすすめコメントを付け他の人に公開出来ます。<br>良いなと思った場所は行きたい場所へ登録が可能です。|
|絞り込み検索機能|行きたい場所,訪問済み,おすすめ場所,それぞれの一覧ページでは登録名,googlemapでの正式名称,住所,レビュー内容,memoでの絞り込み検索が可能です。|
|並び替え機能|行きたい場所,訪問済み,おすすめ場所,それぞれの一覧ページでは登録が新しい順,古い順,googleでの評価が高い順,自分でつけたスコアが高い順での並び替えが可能です。|


## 使用技術
フロントエンド
| 使用技術 | 説明 |
|-----|-----|
| tailwind css 2.2.17| クラス名を考える必要が無くhtmlファイルとCSSファイルを行き来しながらの作業もなくなるなどのメリットがあり、1つのファイルで集中して作業ができ効率よく開発が進むと思い導入しました。 |
| alpine.js 2.8.2|モーダル表示での使用。CDNでの読み込みで導入なので簡単に導入でき、<br>tailwindcssと同じでhtmlファイルに直接処理を書くので管理が楽でtailwindcssとの相性もいいので導入しました。|
| Google Maps API | Geocoding API,Maps JavaScript API,Places APIを使用。検索された場所の緯度,経度,店舗情報を取得するために使用しました。| 


バックエンド
| 使用技術 | 説明 |
|-----|-----|

| rails 6.1.7| rubyのフレームワークとして代表的でrubyと同じで日本語のドキュメントが充実していて、<br>オブジェクト指向を作りながら学習出来ると思い選択しました。|
